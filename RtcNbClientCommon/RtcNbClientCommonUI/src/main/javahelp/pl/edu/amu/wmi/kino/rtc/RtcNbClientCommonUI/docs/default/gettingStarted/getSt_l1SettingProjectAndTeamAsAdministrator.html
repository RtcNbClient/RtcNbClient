Lesson 1: Setting up a project and team as Administrator

In this lesson, you connect to a repository, create a project and team area, and add team members to your team area.

You will connect to the server and create a project area, which defines your project. You will also create a team area which identifies the team members in your project. Team members are known as users. You will define a process for your project area using a predefined process template. The process you select will initialize your configuration with everything you need to try out other components like work items and Rational Team Concert™ source control.

Connecting to a repository for the first time

  1. Create Repository Connection
    • From the menu bar click Window > Services.
    • On the Services tab right-click on RTC Repositories and choose Add new repository
  2. In the Create a Jazz Repository Connectiondialog box
    • Type or paste a server Location URI :
      Note: The Uniform Resource Identifier (URI) for an existing repository. Typically, this is in the form of http:/<server>:<port>/jazz Secure servers require the https:// protocol.
      Note: If the server is running on your computer, the server host name is localhost , and you can select the localhost URI from the dropdown list; for example, https://localhost:9443/jazz. The Location Name defaults to the server host name. You can modify it to simplify or clarify the name.
    • Type the user name and password that you created when you set up the server. .
    • Click Finish
  3. Under created repository connection you will see available project areas. Double click on selected project area to open RTC tab and work work with it.

Creating a project area.

You can create a project area in a repository for managing the project deliverables, team structure, process, and schedule.

To create a project area and team area:

  1. In the Admin Web UI, click the Project Area Management tab.
  2. On the Active Project Areas page, click Create Project Area.
  3. In the Project Name field, type a project name, type Prelude.
  4. To select a process template, under Process, complete one of the following steps.
    • If your team has custom process definitions in the repository, from the list, select a process template.
    • If your team does not have custom process definitions in the repository, click Deploy predefined process templates. From the list, select a process template.
  5. Add an optional summary and description for the project.
  6. If the Jazz™ Team Server is configured to support cross-server communication, you can configure links to project areas on other Jazz Team Servers. To configure linking between project areas, complete the following steps:
    • Under Links , click Add.
    • In the Add Link window, select the link type, target server, and target service providers.
    • Click Finish .
  7. Click Save.

Create a team area

You can create a team area to assign users in particular roles for work on a timeline or a particular set of deliverables. You can create a team area within an existing project area or another team area to establish a team hierarchy. These steps describe how to create a team area within a project area.

  1. In the Admin Web UI, click the Project Area Management tab.
  2. On the Manage Project Areas page, click an active project from the list.
  3. On the project page, in the Team Area Hierarchy pane, click Create Child.
  4. In the Team Area Name field, type a team name, type Team1.
  5. Add an optional summary and description for the team and click Save.

Adding team members to your team area

A team area has several functions. For now, the tutorial will focus on defining team members and adding them to your new team area.

As part of this tutorial you will add three team members:

  • Name=Chris, username=chris, email=chris@example.com, license=Developer
  • Name=Jamie, username=jamie, email=jamie@example.com, license=Developer
  • Name=Build, username=build, email=build@example.com, license=Build System

You can also add other users.

  1. In the Admin Web UI, click the User Management tab.
  2. Select the Active Users page; then click Create User.
  3. Complete the following fields:
    • In the User Name field, type a user name. For example: Chris
    • In the User Id field, type a unique user identifier. For example: chris.
    • In the E-mail Address field, type an e-mail address. For example: chris@example.com
  4. In the Repository Permissions pane, select the appropriate user permissions.
    Note: Passwords and repository permissions are valid only if the server is running a secure connection. When you create a user, the default password is the same as the user identifier. To change the user password, log on to the server as the user and edit the user properties. In the Eclipse client, to edit user properties, in Team Artifacts view, right-click the user repository connection; then click Open My User Editor.
  5. In the Client Access Licenses pane, select the appropriate license type. For example: Rational Team Concert - Developer.
  6. Click Save.
  7. Repeat steps 2 to 6 to add Jamie and Build.
