Modifying permission

Important: Modifying project and teams areas is not available in Netbeans plugin for Rational Team Concert. Following information relates to web user interface.
Permissions are assigned to individual roles at the project level or the team level in a project area, team area, or process template. You can modify the role's permissions for performing operations by using the Permissions tab of the project area or team area. You cannot modify permissions in the process template in the Admin Web UI.
Before you begin
You must be logged in to the Admin Web UI and be a member of the JazzAdmins group.
About this task
To modify the permissions in the project area or team area:
  1. In the Admin Web UI, click the Project Area Management tab.
  2. On the Active Project Areas page, click a project area from the list.
  3. In the Project area editor, click the Permissions tab.
  4. In the Configruation section, click the node for the permissions you want to modify:
    • To modify permissions for operations controlled in the project area, click Project.
    • To modify permissions for operations controlled in the team area, click Team.
    • To modify permissions for operations during a certain type of iteration, expand Iteration Types and click the iteration type.
    • To modify permissions during a certain timeline or iteration, expand Timelines and click the timeline or iteration.
  5. In the Roles section of the Permissions editor, select a role for which you want to modify permissions.
  6. In the Permitted Actions section, expand one of the actions groupings, such as Iteration Plans, Source Control, or Work Items. Select a checkbox to allow the role to perform an action. Clear the checkbox to deny the role permission to perform an action.
  7. Optional: To set the text that is shown when permission to perform an operation is denied, select the operation and enter the text in the Explanation to show when permission denied box.
  8. Click Save to save your changes.
