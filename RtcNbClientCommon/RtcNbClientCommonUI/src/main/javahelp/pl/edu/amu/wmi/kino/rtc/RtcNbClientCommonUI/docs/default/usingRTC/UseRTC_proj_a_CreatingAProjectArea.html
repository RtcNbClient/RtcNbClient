Creating a project area

Important: Modifying project areas is not available in Netbeans plugin for Rational Team Concert. Following information relates to web user interface.
You can create a project area in a repository for managing the project deliverables, team structure, process, and schedule.
Before you begin
You must be logged in to the Admin Web UI and be a member of the JazzAdmins group.
About this task
To create a project area:
  1. In the Admin Web UI, click the Project Area Management tab.
  2. On the Active Project Areas page, click Create Project Area.
  3. In the Project Name field, type a project name.
  4. To select a process template, under Process, complete one of the following steps.
    • If your team has custom process definitions in the repository, from the list, select a process template.
    • If your team does not have custom process definitions in the repository, click Deploy predefined process templates. From the list, select a process template.
  5. Optional: To add a user to the project, complete the following steps:
    1. Under Members or Administrators, click Add.
    2. In the Contributor Selection dialog box, enter a name to search for a particular user, or click Show All.
    3. In the Matching users pane, select a user.
    4. Click Add.
  6. Add an optional summary and description for the project.
  7. If the Jazz™ Team Server is configured to support cross-server communication, you can configure links to project areas on other Jazz Team Servers. To configure linking between project areas, complete the following steps:
    1. Under Links, click Add.
    2. In the Add Link window, select the link type, target server, and target service providers.
    3. Click Finish.
  8. Click Save.
