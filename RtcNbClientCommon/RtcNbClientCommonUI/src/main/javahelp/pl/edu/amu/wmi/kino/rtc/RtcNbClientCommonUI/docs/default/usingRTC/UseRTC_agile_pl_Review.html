Reviewing and charting progress

Important: Plans modifying plans is not available in Netbeans plugin for Rational Team Concert. Following information relates to web user interface.

You can assess your team's progress for a release or for an iteration in several ways. Use the progress report in a plan to compare completed and estimated labor for the whole team. Use the Plans page in a plan to review work items by assigned team member, category, tags, or scheduled time. Use the Charts page in a plan to view a chart of open work items that are planned for this iteration.

To track progress using these methods, your project must have at least one iteration with an iteration plan, and one team area. The most accurate tracking takes place when at least the following details are specified for all work items:
  • Owner
  • "Planned for" iteration
  • Estimated labor in minutes, hours, days, or weeks

Estimated time is calculated using three values in a work item:

  • If the Estimate is the only available value, then that value is used.
  • If a Correction value is available, then that value is used.
  • If a Time Spent value is available and the work item has not been resolved, then that value is subtracted from the estimate or corrected estimate. If the Time Spent value is available and the work item has been resolved, then that value is used.
  • enerally, teams that follow the scrum process use time remaining to specify how much time is left to complete the work item. If Time Remaining value is set, the time spent is derived out of it (time spent=original|corrected estimate - time remaining) to calculate the estimated time.

About this task:

  1. To open a plan, In the Web client, select the Plans link.
  2. Using the progress report in a plan

    The progress report is displayed at the top of the plan. It shows a comparison of completed and expected work hours for the team in this iteration. It also reports the percentage of work items with planned estimates. The width of the progress indicator represents the full duration of the iteration. The length of the green bar shows how much work has been done. A bright green extension shows work completed ahead of plan. A red extension shows work that is behind plan (remaining estimated work exceeds the remaining available hours in the iteration). For more information, hover over the progress report. The percentage of items estimated reflects the quality of the planning. The green and red bars fill the indicator vertically as more items contain work estimates.

  3. Using the Charts page in a plan

    The default report for the Charts page in a plan is the "Open work items by type" report. This report displays open work items that are planned for an iteration and owned by the team area associated with this plan. Open work items are displayed by type (such as defect, enhancement, and task) and charted on an axis representing the weeks in the iteration. The number of open work items should trend towards zero as the project moves toward completion.

    To select a different report for the Charts page:

    1. In the Charts view, click Select Report.
    2. Select one of the available reports, for example, Open vs. closed work items. The chart will display all open and closed work items.
