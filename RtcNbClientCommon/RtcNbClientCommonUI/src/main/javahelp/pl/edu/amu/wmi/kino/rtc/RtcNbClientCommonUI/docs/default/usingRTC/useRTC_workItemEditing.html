Editing work item

A work item is a way of keeping track of the tasks and issues that your team needs to address during the development cycle.

You can edit Work Item using:

  • Netbeans
  • Web UI

Editing work items in Netbeans:

This topic refers to the predefined layout of the work item editor. Administrators can customize the layout. Therefore, the layout that you see might be different
  1. On RTC tab expand a project area , and navigate to a query listed under Work Items folder.
  2. Double-click on a query name or right-click a query and click Open .
  3. Now, work items view will be opened. In this view you see a list of work items. Depends on definened layout of the work item you can see different attributes of work items.
  4. Right-click on work item name that you want to edit and click Edit
  5. A tab with work item editor will be opened. On this view you can choose work item attribute that you want to edit.Below you have details of work item editor.

Use work item editor to:

  • see details of selected work item,
  • update work item.

To see details of work item use one of the following tabs:

  • Overview : display most important information about work item.
  • Links : here you can add attachments and download existing.
  • Approvals : display a list of approvers and enable creating new ones.
  • History: : display a history of editing the work item.

Depends on yours priviliges you can edit different sets of work items attributes. Below you have a description of the most important updates you are able to do as the work item owner. As the work item creator you can edit each attribute of the work item.

  • Before you begin your work : select Start Working from the State property drop-down list.
  • To add comments : if you have a question or comment, in the Discussion section under all comments enter your text and then click Add comment button. All subscribers of the work item receive notification of your comment when you save the work item. When you add a comment, you are added as a subscriber. Another way to collaborate with team members is to enter the @ sign followed by a team member's user ID in a comment or the description
  • To resolve a work item : select Resolve from the State property drop-down list. In the adjacent Resolution property drop-down list, select an entry, such as Fixed, Works For Me, or Duplicate that describes how you resolved the work item. Enter a comment in the Discussion section that further explains your actions. If you resolve a work item as a duplicate, you also need to specify the other work item in the Links section on the Links tab.
  • To add attachments: click link tab on the bottom od work item editor.
Important: Click Save after each update of work item.

Editing work items in Web user interface:

  1. Click the Work Items link in the Web header.
  2. In the left panel click choosen query
  3. Click choosen work item.
  4. Enter or select values for the following fields on the Overview tab (asterisks identify required fields):
    • Summary: A brief headline that identifies the work item.
    • Type: The type of work item, such as Defect, Task, and so on.
    • Severity: An indication of the impact of the work item, such as Minor, Normal, Major, or Critical.
    • Found In: The release that the work item refers to.
    • Filed Against: A category that identifies the component or functional area that the work item belongs to. For example, your project might have GUI, Build, and Documentation categories. Each category is associated with a team area; that team is responsible for responding to the work item.
  5. To attach a file, click Browse and navigate to the file location
  6. To add subscribers, click Add in the Subscribers area. Enter part or all of a user's name, then select the user from the Matching User Names list. Click Add . Subscribers receive E-mail notification whenever someone modifies the work item. When you create a work item, you are added as a subscriber to that work item. To stop receiving notifications, you can remove yourself as a subscriber by selecting your name and clicking Remove.
  7. To link the work item to another work item, in the Links area click the down arrow next to Add and select the option that represents the type of relationship you wish to establish. For example, you can establish a parent-child relationship between work items; mark a work item as a duplicate of another work item; or indicate that one work item is blocked by another work item.
