Managing floating client access licenses

You can install and configure floating client access licenses (CALs) through the administrative Web interface after you determine which server is the floating license server that has the CAL keys installed and which server is the Jazz™ Team Server that assigns the floating CALs.
Before you begin
You must be logged in to the Admin Web UI and be a member of the JazzAdmins group.
About this task
To install floating CALs through the administrative Web interface:
  1. Install the floating CAL keys:
    1. On the floating license server, in the Admin Web UI, click the Server tab.
    2. On the License Key Management page, in the Floating License Server area, click Add .
    3. In the Upload License Files wizard, in the Upload License Activation key field, click Browse and navigate to the floating client access license activation key location.
    4. Select the key.
    5. In the Purchased field, enter the number of licenses purchased.
    6. Click Next .
    7. Optional: On the Connect To Floating Licenser Server page, to configure the floating capabilities on this server, select the Yes, I want to allow users on this server to be able to acquire the floating licenses on this license server check box.
      Note: If you select this check box, you do not need to complete Step 2.
    8. Click Finish .
  2. Configure the Jazz Team Server to point to the floating license server:
    1. On the Jazz Team Server , in the Admin Web UI, click the Server tab.
    2. On the License Key Management page, beside the Floating License Server Property, click Connect .
    3. In the Configure Floating License Server dialog box, in the Floating License Server field, type the path to the server. For example, http://floating-license-server:9080/jazz .
    4. To test the server connection, click Test Connection .
    5. Click Finish . The Jazz Team Server is now configured so that floating CALs can be assigned to users. The floating CALs are automatically checked out from the floating license server when the users require them.
